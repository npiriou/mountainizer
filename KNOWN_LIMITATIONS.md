# Known limitations

- Only the PS2 NTSC-U `SLUS_207.72` disc layout has been fully tested. Other revisions import in clearly marked best-effort read-only mode.
- ISO9660 is supported; UDF-only, multi-track, and unusual multi-extent images are not.
- The 17 playable courses are assembled from observed SDB event, shared-mountain, connector, and sky-area naming. This has been regression-tested across the supplied NTSC-U data, but the game runtime's conditional streaming rules are not completely decompiled.
- Terrain diffuse materials are rendered. Type-1 terrain exposes its exact 432-byte surface/flag/UV/bicubic/sphere/AABB/corner/texture-bank/three-texture/tail layout. Audits 96/97 close all retail texture equations: primary selector 1 emits `ALPHA_1=0x2A` (source replacement), Type-10 selector 8 emits `ALPHA_2=0x81` (`(Cd - Cs) * As`, subtractive lightmapping), and optional-secondary selector 17 emits `ALPHA_2=0x58` (`(Cs - 0) * Ad + Cd`, destination-alpha additive). All 30,644 patches, 623 unique direct-RGBA32 lightmaps, and 3,895 secondary-RID records match those paths. The exact lightmap path is not yet simulated in the normal viewport; the old experimental multiplicative blend produced false colored patch boundaries. Type-0 materials likewise expose their exact three texture slots, zero scratch, bounded state words, animation controls, and optional counted frame-texture table; all observed primary, secondary, and animation-frame references resolve. Normal model rendering still uses the primary texture because the material secondary-slot blend operation, individual state-bit operations, and animation-control units remain unproven. Type-4/5 static fog sprites now have their exact `FOG`/`fog0` texture and `ALPHA=0x44` source-alpha interpolation state. Some PS2 texture formats remain incomplete.
- Splines and AIP navigation paths are decoded as debug curves; Type-11 visibility curtains are decoded as closed, potentially warped quadrilaterals. Curtains expose their exact 208-byte sphere/corners/plane/scratch/AABB/list layout, facing-side test, ascending squared viewer-to-sphere-center candidate score, and nearest-two clipping-plane selection path. Only stripped source-level class/field names and the higher-level streaming policy remain unknown. AIP tagged properties, weighted points, total path lengths, path-distance event intervals, AI respawnability, six-slot section entries, and AI/track position-direction links are framed and runtime-validated. Exact event behaviors, AI metadata word `100`, link-kind/value roles, the second section-entry scalar, and empty-marker gameplay roles are not fully named.
- Type-17 camera-trigger variable records, Replay/InGame flags, volume kinds, radian rotations, exact retail ellipse/box containment transforms, action kinds, nested bound objects, spline payloads, and fill are decoded. Their red display geometry remains a conservative axis-aligned debug box, but queries expose the exact rotated containment semantics; retail ellipses consume all three stored angles while boxes ignore stored rotation Z. The first active-action scalar is identified as blend duration, including the exact `duration == 0 ? 1 : (1/60)/duration` per-frame blend fraction; the complete 76-code Switch Camera remap and Bounded/Spline selection IDs `0x5B/0x5C` are encoded. All five Bounded parameters are executable-proven as camera distance, field-of-view radians, vertical target offset, pitch-offset degrees, and target-forward offset, with exact focus, elevation, and camera-position equations. All four Spline parameters are executable-proven as field-of-view radians, target-forward offset, travel duration seconds, and vertical target offset; its fixed third-point time knots, 101-sample path-speed estimate, 60 Hz arc-length-corrected advance, camera position, and focus equations are encoded. The retail course corpus contains no Spline Camera action, so only its shipped value ranges are unavailable; the factory selection ID `0x5C` and constructed object ID `0x4B` are reported separately. Type-13 sound-trigger binding tables, case-sensitive 64-bit MD5 name-prefix identities, nullable anchor-prop references, distinct shared and per-spatial trigger-info ID channels, the 92-record `WATRIG.ADL` trigger-info catalog, all four runtime spatial-descriptor kinds, their radius/semi-axis/orientation/cone parameters, and all six executable distance/angular falloff equations are decoded. All 33,026 anchored audit occurrences exactly match their prop-name hash. The catalog identifies 52 named ambient-audio records and 39 indexed-bank-sound records; runtime playback proves their two fields are `SoundBankId` and bounds-checked `SoundIndex`, executable config loading names banks 2/5 as `MOUNTAIN`/`CROWD`, resource case 20 proves banks 8/9 are per-track Type-20 resource slots (`TRACK_BANK_0/1`), and kind-4 ID 77 activates a crowd instance. The absent randomized kind-2 path selects four candidates from three explicit bucket widths plus an implicit remainder and ignores word `+0x30`. The 1,128 unanchored identity preimages and individual indexed sound names remain unknown; directional-cone, fixed-falloff, and randomized kind-2 records are executable-supported but absent from the course corpus.
- Type-21 radar/minimap routes expose validated horizontal-world positions, unit tangents, cumulative distances, and Start/Section Boundary/Finish markers. The exact HUD projection/styling remains unknown, and the format intentionally contains no elevation channel. Type-16 is tied to RailMan and exposes packed rail roots, exact six-slot reference sets, generated rail IDs mapped one-to-one after each track's Type-8 spline range, variable rail-program records and bounded reference/directory arrays, per-spline and per-descriptor NonRailMotionPath/SpecialRail/HandplantRail/GrindRail roles with executable-backed surface types, 13-family gameplay-modifier program groups, and multi-routine `LUN` bytecode with exact descriptors and all 43 executable opcode handlers operationally classified. Native calls expose destination slot, native-table index, and argument count; the executable table has 111 entries, with 20 direct names and 20 IDs correlated to 19 subsystems through transitive call-graph strings. Rail-set slot 0 is the sentinel in all 4,757 course records, but the ordered roles of slots 1–5, exceptional global track-0 rail IDs, rail-program kind/control/two-scalar meanings, original source mnemonic of the invalid-result opcode `0x10`, most exact native-function identities, and higher-level program behavior remain unknown. Type-15 is instead the World Painter bank: its 13 serialized IDs are executable-verified as Mix through Danger, with MusicTrigger supported only as runtime ID 0. Record boundaries, per-family sizes, shared blend-control word, 40-byte spatial index, coordinate quantization, quadrant traversal, and empty/record leaves are decoded. Runtime consumers now name every Fog, LightGlow, and ScreenTint property, the Sun RGB triplet, and 17 of Weather's 19 properties. The remaining family-specific names/units and higher-level roles of the Mix/Ambience integer identifiers remain unknown.
- Type-20 `BNKl` slot tables and chained `PT` info sections are decoded through root MIDI note, typed playback envelopes, codec/rate/count/loop/data-offset metadata, and PCM audio. Opcode `0x07` is closed as the RIFF `inst` unshifted/root note with default 60; the shipped overrides span notes 54, 58, 61, 62, 64, and 72. EA's authoring tool proves opcodes `0x09`/`0x19` as a segment count and relative pointer to `(duration, volume)` pairs, with durations in hundredths of a second and volumes from 0–127; release-segment index and initial-volume defaults are also preserved. Retail code proves signed-negative durations saturate to `0x7FFFFFFF`, accumulates an exact 16.16 volume slope while decrementing the duration, advances at zero, and jumps forward to the release segment on release. It also proves `0x01/02` are inclusive velocity bounds and `0x03/04` are inclusive MIDI-note bounds; every matching `FE`-chained section creates a voice, so shipped default-range chains are simultaneous layers rather than random alternatives. The version-3 MicroTalk decoder implements 432-sample multipulse CELP/RELP frames, relative loop-substream resets, per-frame PCM correction blocks, bounded declared-sample output, and PCM16/WAV creation. All 404 unique and 668 assembled sections decode to their declared lengths. Interactive mixing, envelope application, and playback UI remain unimplemented. Type-22 avalanche streams expose world-space block origins and 30 Hz records whose signed translations and axis-angle rotations are cumulative interval increments while scale is linearly interpolated. Retail code proves the endpoint clamp and schedule behavior: disk `(block, frame)` pairs become runtime `(frame, block)` pairs and all ordered records with `frame <= elapsedSeconds * 30` dispatch, including duplicates. Timed identities are snapshots of target objects' packed self-references; the loader deliberately discards them, classifying EBC3 `0:260` as a non-restored runtime identity rather than a missing Type-3 dependency. The concrete runtime class behind that identity and full avalanche rendering/physics simulation remain unknown.
- Named non-visual gameplay props (reset planes, volumes, trigger meshes, ride-state meshes, effects, streaming markers, proxies, and collision walls) are recognized, hidden by default, and independently toggleable by category. Dedicated Type-12 version-1 collision topology and version-3 sphere trees are decoded for inspection, but they are not rendered or exported. Version 1 now exposes its exact flat acceleration structure: one bit-identical min/max AABB for each consecutive group of at most ten triangles, plus exact alignment and runtime pointer-scratch regions. Runtime tracing proves Type-12 stores no surface/material value; the hit attribute is copied later from a separate owning-collider sub-object record. Its `-1` default resolves to Packed Snow (0), and gameplay consumers plus executable names establish the shared 0–18 surface enum, including Ridable Rock, Pavement, and Wipeout Rock. For Type-3 props the owner is one shared 24-slot profile; exhaustive executable store/xref analysis proves it is constructor-only and all slots remain `-1` in retail, so there is no missing dynamic override writer. Version 3 exposes the exact signed-control RLE stream, dense octree address space, eight-bit child-selection masks, per-level `1, 8, 64, ...` strides, referenced nodes/links, correction vector, radii, and retained metadata comprising a vector plus an exactly symmetric positive-definite 3x3 matrix with its verified symmetric inverse. An executable-wide record-pointer survey proves the retained metadata is copied but never consumed after construction, closing its retail role as unused; only its original offline authoring label is unrecoverable. Type-4 particle models expose bounds and repeated position/color/size elements, while paired Type-5 emitters, Type-6 lights, and Type-7 halos expose validated transforms or positions in the inspector. Type-6 now exposes runtime-proven Intensity/SelectionWeight, spot cone cosines, signed falloff exponents, the exact point/spot selection and RGB-contribution equations, and the invariant tail marker. Directional/Ambient rendering outside the decoded local-light path and original source names for flag `0x100` and marker `0x4039` remain unknown. Type-7 visual modes, cubic coverage, octree queue/batching, screen-space occlusion probe/readback, double-buffered visibility writeback at `+0x40/+0x44`, RGB normalization, final render scales, named `SHALO/shal` and `MHALO/mhal` textures from `data/textures/effects.ssh`, exact source-alpha additive GS blending, and both serialized pointer-token families are validated. The exact per-track `entryBase + 8 × resourceId` relation describes serialized `+0x44`, which runtime later overwrites with visibility state; `+0x00/+0x04` are likewise overwritten by intrusive links. Only original pointer-token address space/producer and framing/mode source names remain unavailable; they are offline provenance labels rather than missing retail render behavior. Experimental point markers are disabled in the normal viewport because they are not the original effects; particle/halo behavior and full light rendering are not yet simulated. Type-18 is now confirmed as an 18-slot `cSSXScriptEngine` object-transform lookup rather than a playback timeline; every slot has an executable command route and missing slots retain neutral initialized outputs. Exact source-level names for commands 19-39 and the higher-level script sequences remain unknown. Type-15 World Painter effects, the remaining Type-16 RailMan/LUN semantics, Type-20 playback, and decoded Type-22 avalanche animation data remain unrendered and unsimulated.
- Type-3 no longer stops at the 160-byte fixed header: its complete variable tail is decoded as PS2 DMA/VIF relocation subprograms, including hardware tag fields, model/instance `REF` roles, `RET` framing, both SPR rewrite forms, bounded source blocks, framed VIF commands, and the terminal placeholder rewritten to `MSCAL`. All 115,249 source blocks follow the exact `STCYCL` / signed TOPS-relative `UNPACK V4-5` / `STMASK` / placeholder / `ITOP 1` sequence and contain 4,112,542 per-instance RGBA5 vertex colors. Every array has the same count as, and begins immediately after, the matching model packet's terminal per-vertex array. The fixed header exposes its serialized-zero runtime prefix, transform, authored bounding sphere, world AABB, exact self-reference, repeated track byte, exact texture-bank subchunk ID, model reference, scale, an unused signed `-1` retail sentinel at `+0x90`, a wind-deformation halfword at `+0x92`, runtime scratch, and the extension pointer. All 41,113 unique assets partition exactly into structural and referenced source quadwords, and all 53,766 model-material references select a Type-9 texture from the serialized subchunk. The wind halfword is `1` on 1,203 simple visual instances (1,198 vegetation-named and 48 models shared with the disabled state); executable `0x0037e4d0` proves that it gates an in-place render-matrix deformation callback. A complete access survey finds no Type-3 read or write of `+0x90`. Instance `+0x88` is serialized as zero and receives the single shared loader collider-profile pointer; the collision query's signed attribute halfword lives in that external object and uses the shared 0–18 `SsxSurfaceType` numbering, while the immutable profile's `-1` default is normalized to Packed Snow (0).
- The coordinate conversion boundary maps the observed SSX 3 Z-up data into Mountainizer's Y-up scene space as `(x, y, z) → (x, z, -y)`. Units, canonical winding, and handedness still need wider revision/platform validation.
- Terrain and decoded props use triangle-accurate viewport picking after a cached bounds broad phase. Debug-only splines, trigger boxes, and visibility curtains still use their decoded bounds. Detachable/persistent docking is not implemented.
- OBJ export includes terrain and decoded prop geometry, an MTL library, and decoded diffuse PNG textures. Lightmaps and material flags are not exported; glTF is not implemented.
- Audit 72 correction for Type 7: texture identities, color packing, and blending are now proven. Modes `0x10/0x40` use `SHALO/shal`, mode `0x20` uses `MHALO/mhal`, both from `data/textures/effects.ssh`; final submission packs normalized RGB and the occlusion fraction as GS RGBA with scale 128. Internal selector 7 compiles to GS `ALPHA=0x48`, the exact source-alpha additive equation `(Cs - 0) × As + Cd`. Remaining unknowns are offline/source labels rather than the retail render equation.
- Audit 94 correction for Type 6: there is no separate missing Directional/Ambient course-light renderer. Loader `0x003AB918` calls predicate `0x003AACA8`, which admits only unflagged Spot/Point records; all Directional, Ambient, and flag-`0x100` records are rejected before internal type assignment and world-index insertion. Only the original editor-facing names for flag `0x100` and tail marker `0x4039` remain unavailable.
- Audit 95 correction for Type 21: the serialized unit vector is a lateral normal, not a route tangent. Runtime `0x00210618` chooses the greatest sample distance not exceeding the rider's course distance, clamped at the endpoints; `0x00210820` computes `dot(riderPosition - samplePosition, lateralNormal)`. The 1P radar at `0x0020EDA0` uses a 10,000-unit window centered on the local rider and the retail textures `radarline`, `chkstart` (kinds 0/2 Start/Finish), and `chkpt` (kind 1 Checkpoint). The older Type-21 limitation claiming projection/styling is unknown is superseded; only absent elevation and original editor/source labels remain.
- Audit 75 closes the Type-8 spline runtime record: exact segment chains/owner fixups, arc and cumulative lengths, cubic distance-to-parameter mapping, spatial position/tangent/curvature coefficients, bounds, and total-length behavior are executable-verified across 2,659 resources / 11,258 segments. Only segment prologue word `+0x00` lacks a safe source-level name; Type-16 already supplies each spline's gameplay role and surface metadata.
- Audit 76 corrects Type 11 from a supposed 184-byte cubic into the exact 208-byte visibility-curtain record. All 167 unique resources contain four authored corners, a sphere, plane, 64-byte zero scratch, AABB, loaded flag, and zero list-pointer/trailing scratch; retail keeps them in an intrusive manager list. Nine resources intentionally have a fourth corner off the plane defined by the first three, so the decoder preserves a warped quadrilateral. Audit 79 closes the selector: the viewer must be on the non-positive side of the stored plane, candidates are sorted by ascending squared XYZ distance to the stored sphere center (radius excluded), and at most the nearest two planes are installed.
- Audit 77 closes the Type-0 serialized framing. All 2,575 unique resources fit an exact 20-byte base plus optional counted texture-RID frame table: 2,550 base records, 23 two-frame records, and two five-frame records. The parser now preserves three texture slots, three zero scratch halfwords, two state words, two animation controls, and every frame RID instead of ignoring extended tails. Across 3,373 assembled-course occurrences, all 3,540 distinct-per-material texture references resolve. Remaining questions are the state-bit operations, animation-control units/roles, and secondary-texture blend behavior, not missing bytes.
- Audit 78 closes the Type-1 serialized framing and corrects the former 428-byte partial interpretation to the exact 432-byte retail record. All 30,644 unique resources preserve the 16 homogeneous coefficient lanes and final tail words; reconstructed controls exactly identify the control-net AABB and four corner vectors. The first halfword maps to the shared 0-18 surface enum, and the renderer proves primary/lightmap/secondary texture RIDs plus bounded texture/render flags. Across 34,500 course occurrences all 38,395 primary/secondary diffuse references resolve. Remaining questions are source names and combine/state behavior, not missing bytes.
- Audit 96 closes Type-1's optional secondary combine/state behavior. Renderer `0x0038CE20` and helper `0x0038D168` create a separate context-2 draw; compiler `0x00363F9C..0x00363FE4` maps render bit `0x0040` to alpha selector 17 and GS `ALPHA_2=0x58`, or `(Cs - 0) * Ad + Cd`. The dormant fallback is selector 2 with `FIX=128`, plain `Cs + Cd`. The remaining Type-1 combine unknown is specifically the always-present primary/lightmap path.
- Audit 97 supersedes Audit 96's remaining Type-1 combine unknown. Parent setup `0x0038BCB4..0x0038BD78` installs primary selector 1 / `ALPHA_1=0x2A` (`Cs`) and lightmap selector 8 / `ALPHA_2=0x81` (`(Cd - Cs) * As`). All 623 unique Type-10 resources are exact RGBA32 atlases totaling 6,984,704 pixels, with raw PS2 alpha spanning 0..255 and `0x80=1.0`. Type-1's remaining unknowns are source/editor labels and invariant framing roles, not texture equations.
- Audit 98 supersedes the older Type-0 secondary-slot/animation-control limitations. Type-0 contains one primary Type-9 RID plus an optional counted table of randomized texture variants, not three texture slots. Material `+0x02` bit `0x0008` selects the exact primary equation: clear uses selector 1 / `ALPHA_1=0x2A` (`Cs`, opaque replacement); set uses selector 5 / `ALPHA_1=0x44` (`(Cs - Cd) * As + Cd`, source-alpha interpolation). All 6,203 Type-9 assets have a zero base dispatch mask, so these are the only shipped outcomes. Registration initializes the per-use frame index as unsigned RNG output modulo the shared model frame count. Material `+0x04` is exactly a DMAC ID-5/QWC-0 CALL-target address adjustment; only the offline producer reason/name for its invariant `+0x0000ffff` bias remains unavailable. Other remaining Type-0 unknowns are the non-bit-3 state source names/operations.
- Development builds require the .NET 8 Desktop Runtime. Published releases are verified as self-contained Windows x64 single-file executables; installer packaging and code signing are not implemented.
