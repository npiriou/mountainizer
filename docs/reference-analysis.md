## Confirmed pipeline from the supplied user-local disc

Type-14 constructors at `0x0026b310` and `0x0026b410` allocate 60-byte TrackPath and 64-byte AIPath runtime records and feed both through the common reader at `0x0026a180`. That reader proves the apparent fixed headers are counted `(kind, byte length, payload)` properties. Track property `0` is loaded as the distance-to-finish scalar; AI property `101` is a boolean respawn flag (also enforced by closest-path selection at `0x0026b050`), while AI property `100` remains conservatively named metadata. Helper `0x0026ac48` sums point weights as total path length. Event loading maps serialized codes through the 21-entry table at `0x004453d0`; runtime interval tests consume their two floats as start/end path distances. Tail loader `0x0026b880` maps raw link kinds through `{-1,0,1}`, preserves a position and unit direction, and replaces the final two indices with pointers to 64-byte AIPath and 60-byte TrackPath records. Audit 43 validates all 84 assembled tables, 6,158 event intervals, and 261 links; see `artifacts/type14-runtime-navigation.md`.

Type-12 version-3 loader `0x00327da8` proves the fixed metadata contains two vectors and two 3x3 matrices rather than four cover points plus one matrix. Across all 300 unique records, the matrices are exactly symmetric, positive-definite, and mutually inverse. An executable-wide runtime-record pointer survey proves that the loader retains the second vector and matrix pair at `+0x2c..+0x7f`, but no decoder or query path reads or writes that range after construction; they are therefore exposed conservatively as retail-unused retained metadata rather than an inferred matrix center. Decoder `0x0032b620` expands a signed-control literal/run RLE stream, while traversal `0x0032cbf8` treats every decoded byte as an eight-child mask and advances with the declared `1, 8, 64, ...` level strides. Audit 67 expands 89,235 packed bytes into 1,250,124 dense octree bytes with zero structural violations and records the copied-but-unread runtime status; see `artifacts/type12-v3-runtime-sphere-octree.md`.

Type-6 light selection at `0x002f5d30` and local RGB contribution at `0x0038a6a8` prove that record `+0x14` is Intensity while `+0x18` is a top-N SelectionWeight absent from the RGB path. Both consumers use a hard `+0x1c` range, no inverse-distance attenuation inside 100 units, signed integer powers of `100/distance` outside it, and a separate linear fade from 3,750 to 5,000 units for 5,000-range lights. Spot fields `+0x5c/+0x60` are inner/outer cone cosines; `+0x64/+0x65` are distance/angular exponents; the cone shoulder linearly interpolates from zero at the outer cosine to `innerCosine^exponent`, then uses `dot^exponent` inside. The RGB scalar is clamped to 0–5, while the selection path multiplies the unclamped scalar by SelectionWeight. Loader `0x003ab918` calls Type-6 vtable predicate `0x003aaca8`, whose exact equation admits only unflagged Spot/Point records; Directional, Ambient, and flag-`0x100` records are rejected before internal type assignment or world-index insertion. All 1,213 serialized spot records have ordered cone cosines, and all 1,962 Type-6 records carry tail marker `0x4039`; see `artifacts/type6-runtime-light-equations.md` and `artifacts/type6-runtime-admission.md`.

Type-7 resource case `0x003aba70` registers each halo's serialized `+0x28..+0x3f` cube with the spatial index and overwrites header `+0x08` with internal resource type 8. Type-3 loader `0x003ac2a8` resolves the packed Type-2 model, injects its loader-owned collision-metadata pointer at instance `+0x88`, and calls `0x003ac358` to relocate the always-`0xA0` extension pointer. Walker `0x003ac508` proves the variable tail is a PS2 DMA/VIF relocation program: tag bits `28..30` are the hardware DMA IDs, the low halfword is QWC, word 1 carries address/SPR, ordinary steps pair model-relative and instance-relative `REF` tags, and `RET` ends each model-derived subprogram. Every instance source block ends in a `DEADBEEF` slot plus VIF `ITOP 1`; the loader replaces the placeholder with VIF `MSCAL` command `0x14` for VU entries `0x44e`/`0x459`. Every block frames completely as NOP padding around `STCYCL`, one signed TOPS-relative `UNPACK V4-5`, `STMASK`, the placeholder, and `ITOP 1`. Pairing each source `REF` with its preceding model `REF` identifies the V4-5 payload as the model packet's contiguous per-vertex RGBA5 color array: all 115,249 pairs have identical element counts and adjacent VU destinations. The 4,112,542 values use full 5-bit RGB lanes and a one-bit alpha lane; only 19,354 alpha-zero values occur, concentrated in snow-stream, translucent-sheet, and searchlight assets. Constructor `0x003aa2f0` shows that injected `+0x88` is the single shared 0x130-byte profile at `0x00508f50`: a 16-byte header plus 24 twelve-byte subobject slots initialized to floats `1000.0`/`0.5`, halfword zero, and signed collision attribute `-1` (no override). What appeared as a 32-bit flags word at `+0x90` is actually two halfwords: an unused invariant `-1` retail sentinel at `+0x90` and a `0/1` wind-deformation switch at `+0x92`; a complete access survey finds no Type-3 consumer for the former. ## Incompatibilities and risks

- Replanetizer targets different games and PS3 data; only UI/renderer organization transfers conceptually.
- The decompilation and rehost model live runtime data, while Mountainizer reads compressed disc resources. Addresses and in-memory pointers are not on-disk offsets.
- Existing SSX tools commonly combine parsing with extraction or serialization and trust counts. Mountainizer separates raw framing, scene conversion, rendering, and export and rejects unsafe sizes.
- SDB location-to-group association is non-obvious: the tested data associates the next location's subchunk count with the current extractor-style span. The chunk/subchunk tables and Type-3 track placement now corroborate this layout, but another game revision is still needed for cross-revision validation.
- Existing names such as “patch,” “MDR,” and resource type meanings are working terminology, not guaranteed original EA names.
